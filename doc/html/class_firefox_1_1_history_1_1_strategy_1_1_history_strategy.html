Подробное описание

Стратегия чтения и записи истории посещений Firefox.

Класс HistoryStrategy предназначен для извлечения истории посещений веб-страниц из таблицы moz_places базы данных профиля Firefox.

Данные считываются пакетами, преобразуются в структуру History и сохраняются в результирующую базу данных.

Преобразование временной метки последнего посещения выполняется средствами SQLite.

Создаёт таблицу history в результирующей базе данных.

Таблица предназначена для хранения истории посещённых веб-страниц. Каждая запись соответствует одному URL из таблицы moz_places. Дополнительно создаётся индекс по полю url для ускорения поиска.

◆ execute()

None Firefox.History.Strategy.HistoryStrategy.execute ( self,

Запускает полный цикл обработки истории посещений.

Аргументы
executorПул потоков (не используется напрямую в данной стратегии).

Последовательно:

  1. Создаёт таблицу history;
  2. Считывает историю посещений пакетами;
  3. Записывает данные в результирующую БД;
  4. Создаёт служебные таблицы и метаданные;
  5. Сохраняет результирующую базу данных.

◆ read()

Generator[list[History], None, None] Firefox.History.Strategy.HistoryStrategy.read ( self)

Считывает историю посещений из базы данных Firefox.

Метод выполняет выборку данных из таблицы moz_places базы данных профиля Firefox.

Чтение осуществляется пакетами по 500 записей за итерацию.

Поле last_visit_date преобразуется средствами SQLite из микросекунд Unix Epoch в читаемый формат datetime.
