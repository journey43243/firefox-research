Подробное описание

Стратегия чтения и записи сохранённых паролей Firefox.

Стратегия использует сервис PasswordService для извлечения и дешифрования сохранённых учётных данных браузера Firefox, после чего выполняет пакетную запись результатов в выходную базу данных.

Предупреждения
Стратегия обрабатывает конфиденциальные данные (логины и пароли). Рекомендуется использовать исключительно в доверенной среде.

◆ createDataTable()

Firefox.Passwords.Strategy.PasswordStrategy.createDataTable ( self)

Создаёт таблицу passwords в результирующей базе данных.

Таблица предназначена для хранения сохранённых паролей Firefox и содержит URL сайта, имя пользователя, пароль и идентификатор профиля.

Для предотвращения дублирования записей используется ограничение UNIQUE.

◆ createHeadersTables()

Firefox.Passwords.Strategy.PasswordStrategy.createHeadersTables ( self)

Создаёт и наполняет таблицу Headers для паролей Firefox.

Таблица Headers содержит метаданные столбцов таблицы passwords и используется для корректного отображения и экспорта данных во внешних интерфейсах и отчётах.

◆ execute()

None Firefox.Passwords.Strategy.PasswordStrategy.execute ( self,
list[Task] tasks )

Запускает полный цикл обработки паролей Firefox.

Последовательно:

  • создаёт таблицу passwords;
  • считывает и дешифрует пароли пакетами;
  • создаёт асинхронные задачи записи для каждого пакета;
  • формирует служебные таблицы и метаданные;
  • сохраняет результирующую базу данных на диск.
Аргументы
tasksСписок asyncio-задач, в который добавляются задачи записи для последующего ожидания.

◆ read()

Generator[list[tuple[str, str, str, int]], None, None] Firefox.Passwords.Strategy.PasswordStrategy.read ( self)

Считывает и дешифрует сохранённые пароли профиля Firefox.

Выполняет извлечение и дешифрование всех сохранённых паролей с использованием сервиса PasswordService.

Полученные данные группируются в пакеты фиксированного размера (500 записей) для последующей записи.

Возвращает
Генератор пакетов кортежей вида: (url, user, password, profile_id).

Запись выполняется в рамках транзакции с использованием INSERT OR IGNORE для предотвращения дублирования записей.
