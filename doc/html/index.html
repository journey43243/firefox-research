О проекте

Проект "firefox-research" — это набор модулей и утилит для извлечения, нормализации и сохранения артефактов браузера Mozilla Firefox из пользовательских профилей. Цель проекта — предоставить удобную, воспроизводимую и расширяемую инфраструктуру для сбора данных (история, закладки, загрузки, расширения, пароли, профили и т.д.) и сохранения их в результирующие SQLite-базы для последующего анализа.

Быстрый старт

  • Установите зависимости: pip install -r requirements.txt
  • Создайте папки Cases и Logs в корне проекта (если ещё нет).
  • Запустите анализ кейса: python Run.py.

Архитектура и особенности

Проект построен вокруг набора независимых стратегий (Strategy), каждая из которых отвечает за чтение конкретной части данных Firefox и запись результатов в отдельную результирующую SQLite-базу. Общие интерфейсы и механизмы:

  • Modules/Firefox/Parser.py — точка входа для модуля Firefox; координирует инициализацию стратегий и параллельное выполнение.
  • Modules/Firefox/interfaces/Strategy.py — абстрактные классы StrategyABC и Metadata, единый контракт для всех стратегий (read/write/execute).
  • Для каждой логической области данных создаётся отдельная стратегия и файл с реализацией (Profiles, History, Bookmarks, Downloads, Cookies, Extensions, Passwords).

Модули Modules/Firefox

Ниже — краткое описание основных модулей, находящихся в Modules/Firefox.

  • Parser
    • Файл: Modules/Firefox/Parser.py
    • Описание: Оркестратор выполнения стратегий. Находит профили Firefox, создаёт объекты Metadata и запускает стратегии (History, Passwords, Bookmarks, Downloads, Extensions и т.д.) в пуле потоков.
  • Profiles
    • Файл: Modules/Firefox/Profiles/Strategy.py
    • Описание: Читает profiles.ini, извлекает абсолютные пути профилей и сохраняет их в результирующую базу. Эти пути служат источником для всех дальнейших стратегий.
  • History
    • Файл: Modules/Firefox/History/Strategy.py
    • Описание: Извлекает записи из moz_places, преобразует метки времени и сохраняет историю посещений в отдельную таблицу history.
  • Bookmarks
    • Файл: Modules/Firefox/Bookmarks/Strategy.py
    • Описание: Считывает закладки (moz_bookmarks + moz_places), формирует структуру записей и сохраняет их в результирующую таблицу bookmarks.
  • Passwords
    • Файл: Modules/Firefox/Passwords/PasswordService.py
    • Описание: Работа с NSS (nss3) — инициализация профиля, проверка мастер-пароля и дешифрование сохранённых логинов (logins.json / signons.sqlite). Возвращает расшифрованные пары (url, user, password).
  • Downloads
    • Файл: Modules/Firefox/Downloads/Strategy.py
    • Описание: (стратегия для извлечения записей загрузок из аннотаций/places).
  • Cookies
    • Файл: Modules/Firefox/Cookies/Strategy.py
    • Описание: (стратегия для извлечения и сохранения cookies Firefox).
  • Extensions
    • Файл: Modules/Firefox/Extensions/Strategy.py
    • Описание: Обрабатывает extensions.json профиля — извлекает список установленных дополнений, их метаданные и состояние.
  • interfaces
    • Путь: Modules/Firefox/interfaces/Strategy.py
    • Описание: Общая инфраструктура (контракт StrategyABC, Metadata), вспомогательные методы для создания интерфейса записи SQLite и формирования метаданных.

Результаты работы

Для каждой стратегии создаётся отдельная результирующая база SQLite в каталоге кейса. Каждая база содержит:

  • основную таблицу с извлечёнными артефактами;
  • Info — служебная таблица с метаданными (имя модуля, время выполнения, производитель);
  • Headers — описание колонок для визуализации/экспорта.

Замечания и рекомендации

  • Модуль Passwords использует нативную библиотеку NSS; на целевой платформе должна быть доступна nss3 (nss3.dll на Windows). При развертывании убедитесь, что путь до библиотеки корректен или библиотека установлена в системе.
  • Для корректной работы Парсера требуется доступ к файлам профиля Firefox (папки профилей в APPDATA%\Mozilla\Firefox).
  • Логирование и результаты записываются в папки Logs и Cases.
